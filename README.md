# ADHD 药物管理助手

一个面向 ADHD 用户的轻量级药物记录网站原型，用来记录服药时间、药后表现、睡眠情况和其他药物影响，帮助用户长期观察自己的个人规律。

## 项目背景

很多 ADHD 用户会遇到这些问题：

- 忘记自己什么时候吃了药
- 很难回忆吃药后到底是更专注了，还是更焦虑了
- 不容易判断服药时间、睡眠和当天状态之间的关系
- 同时服用其他药物时，很难回顾哪些因素影响了当日表现

这个项目希望用一个足够轻量、低负担、适合手机使用的网页，把这些信息记录下来，并帮助用户更直观地回顾趋势。

## 核心功能

- 记录 ADHD 药物名称、服药时间、剂量
- 记录其他药物或补充剂
- 记录吃药后表现，按多个类别折叠选择
- 记录睡眠情况，包括入睡困难和原因
- 使用滑块快速记录清醒度、焦虑程度、情绪稳定度、执行状态
- 自动生成最近规律摘要
- 历史记录支持查看与编辑
- 使用浏览器 `localStorage` 做本地保存

## 交互设计亮点

- 表单尽量减少打字，多用点选、滑块、标签
- “吃药后表现”做成可折叠结构，减少页面长度
- “入睡原因”只在选择“入睡困难”时显示
- 历史记录可以一键回填到表单继续修改
- 页面适配手机和桌面端

## 适用场景

- 个人长期记录 ADHD 药物反应
- 观察服药时间与专注度、情绪、睡眠之间的关系
- 作为和医生沟通时的辅助记录工具

## 本地运行

直接打开 `index.html` 即可预览，或在项目目录启动一个本地静态服务：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## 当前版本说明

这是一个前端原型版本，重点在于验证产品结构和交互方式。

当前限制：

- 数据只保存在当前浏览器
- 不支持账号登录
- 不支持云端同步
- 不提供医疗建议，仅用于个人记录和观察

## 后续可扩展方向

- 接入数据库，实现多设备同步
- 增加账号系统
- 增加图表分析和时间段对比
- 支持导出记录
- 生成更适合医生查看的报告

## 医疗边界说明

本项目是自我观察工具，不替代医生建议，不用于诊断、处方或自行调整药物方案。

任何停药、换药、加药、减药、改剂量等决定，都应与医生沟通。
